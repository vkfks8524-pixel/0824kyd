Cloudflare에서 도메인이 확인 중으로 오래 남을 때 점검할 항목

작성일: 2026년 7월 12일 · 예상 읽기 시간: 5분

Cloudflare Pages에 도메인을 연결했는데 “확인 중” 상태가 오래 보이면 불안할 수 있습니다. 하지만 대부분은 네임서버 전파, DNS 레코드, SSL 인증서 확인 과정에서 생기는 정상 대기입니다.

1. 네임서버가 Cloudflare로 바뀌었는지 확인

도메인을 구매한 업체에서 Cloudflare가 안내한 네임서버 두 개를 정확히 입력했는지 확인합니다. 기존 네임서버가 함께 남아 있으면 확인이 늦어질 수 있습니다.

2. Pages 사용자 지정 도메인에서 추가했는지 확인

Cloudflare DNS 메뉴에서 레코드만 수동으로 만든 것과 Pages 프로젝트의 사용자 지정 도메인 메뉴에서 연결한 것은 다릅니다. Pages 프로젝트 안에서 도메인을 추가해야 인증이 정상적으로 진행됩니다.

3. 루트 도메인과 www를 구분하기

kyd.kr과 www.kyd.kr은 서로 다른 호스트입니다. 둘 다 사용할 계획이면 둘 다 사용자 지정 도메인에 추가하고, 대표 주소를 하나로 정해 리디렉션을 설정합니다.

4. SSL 인증서 발급 기다리기

DNS가 맞아도 HTTPS 인증서가 준비되기까지 시간이 걸릴 수 있습니다. 이때 도메인을 삭제하고 다시 추가하기보다 상태가 바뀌는지 기다리는 편이 낫습니다.

5. 24시간 후에도 그대로일 때

하루가 지나도 확인 중이라면 네임서버 오타, DNSSEC 설정, 잘못된 CNAME, 다른 서비스와의 충돌을 확인합니다. 특히 DNSSEC를 사용하던 도메인은 DS 레코드가 남아 있는지 볼 필요가 있습니다.

정리

확인 중 상태는 무조건 오류가 아닙니다. 네임서버, Pages 연결 위치, 루트와 www 구분, SSL 인증서, DNSSEC 순서로 차분히 확인하면 대부분 해결 방향을 찾을 수 있습니다.
