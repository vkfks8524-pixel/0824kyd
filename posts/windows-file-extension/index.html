파일 관리

Windows에서 파일 확장자를 표시해야 하는 이유와 설정 방법

작성일: 2026년 7월 12일 · 예상 읽기 시간: 8분

파일 이름 뒤의 .pdf, .jpg, .exe 같은 부분을 확장자라고 합니다. 확장자를 보이게 해두면 파일 종류를 더 정확하게 확인할 수 있습니다.

1. 확장자가 중요한 이유

Windows는 기본 설정에서 알려진 파일 형식의 확장자를 숨길 수 있습니다. 이 경우 안내문.pdf처럼 보이는 파일이 실제로는 안내문.pdf.exe일 수도 있습니다. 확장자를 숨기면 사용자는 마지막 .exe를 보지 못하고 PDF라고 착각할 수 있습니다.

모든 파일이 위험한 것은 아니지만, 출처가 불분명한 파일을 열 때 확장자를 확인하는 습관은 안전에 도움이 됩니다.

2. Windows 11에서 확장자 표시하기

  1. 파일 탐색기를 엽니다.
  2. 상단 메뉴에서 보기를 선택합니다.
  3. 표시 메뉴로 들어갑니다.
  4. 파일 확장명 항목을 켭니다.

3. Windows 10에서 확장자 표시하기

  1. 파일 탐색기를 엽니다.
  2. 상단의 보기 탭을 선택합니다.
  3. 파일 확장명 체크박스를 켭니다.

4. 자주 보는 확장자

  • .pdf: 문서 파일
  • .jpg, .png: 이미지 파일
  • .docx, .xlsx: Word, Excel 문서
  • .zip: 압축 파일
  • .exe, .msi: 실행 또는 설치 파일

5. 파일 이름을 바꿀 때 주의할 점

확장자를 표시한 뒤 파일 이름을 바꿀 때는 마지막 확장자를 실수로 지우지 않도록 주의합니다. 예를 들어 사진.jpg에서 .jpg를 지우면 Windows가 어떤 프로그램으로 열어야 할지 헷갈릴 수 있습니다.

정리

확장자를 표시하는 것은 작은 설정이지만 파일을 안전하게 확인하는 데 도움이 됩니다. 특히 이메일 첨부파일이나 메신저로 받은 파일을 열기 전에는 파일 이름 전체를 확인하는 습관을 들이는 것이 좋습니다.

확장자를 봐야 하는 이유

Windows는 기본 설정에서 알려진 파일 형식의 확장자를 숨길 수 있습니다. 이 상태에서는 문서처럼 보이는 파일이 실제로 실행 파일인지 알아보기 어렵습니다.

예를 들어 안내문.pdf.exe 같은 파일은 확장자가 숨겨져 있으면 안내문.pdf처럼 보일 수 있습니다. 파일 확장자를 표시하면 이런 위험을 더 빨리 알아차릴 수 있습니다.

설정 후 확인할 파일

확장자를 표시한 뒤 다운로드 폴더와 바탕화면의 파일을 확인합니다. docx, xlsx, pdf, jpg, png 같은 문서나 이미지 파일과 exe, msi, bat, cmd, js 같은 실행 가능 파일을 구분해봅니다.

압축 파일도 조심해야 합니다. zip 파일 자체는 흔하지만, 압축을 푼 뒤 안에 실행 파일이 들어 있으면 바로 실행하지 말고 출처를 다시 확인해야 합니다.

초보자에게 권하는 습관

파일을 열기 전에 아이콘보다 확장자를 먼저 보는 습관을 들이면 실수를 줄일 수 있습니다. 아이콘은 비슷하게 꾸밀 수 있지만 확장자는 파일의 실제 형식을 알려줍니다.

업무나 학교에서 받은 파일이라도 확장자가 예상과 다르면 보낸 사람에게 다시 확인합니다. 특히 급하게 처리하라는 문구와 함께 온 파일은 더 조심하는 것이 좋습니다.

확장자와 아이콘의 차이

파일 아이콘은 사용자를 속이기 위해 비슷하게 꾸밀 수 있습니다. 반면 확장자는 Windows가 파일을 어떤 방식으로 열지 판단하는 중요한 단서입니다. 그래서 아이콘보다 확장자를 먼저 보는 것이 안전합니다.

예를 들어 이미지 아이콘처럼 보여도 확장자가 exe라면 실행 파일입니다. 문서 아이콘처럼 보여도 js나 bat라면 스크립트 파일일 수 있습니다. 확장자를 표시하면 이런 차이를 바로 볼 수 있습니다.

확장자를 바꾸면 생기는 오해

파일 이름 끝부분을 직접 바꾼다고 파일 형식이 실제로 변하는 것은 아닙니다. 예를 들어 jpg를 pdf로 이름만 바꿔도 내용이 PDF 문서가 되는 것은 아닙니다. 오히려 파일이 열리지 않을 수 있습니다.

확장자를 변경해야 할 때는 원본을 복사해둔 뒤 진행하는 것이 좋습니다. 중요한 파일을 잘못 바꾸면 어떤 프로그램으로 열어야 하는지 헷갈릴 수 있습니다.

마지막 점검 포인트

확장자를 표시해두면 메일 첨부파일과 다운로드 파일을 확인할 때 특히 도움이 됩니다. 보낸 사람이 문서라고 했는데 실제 확장자가 실행 파일이면 열기 전에 멈출 수 있습니다. 이런 작은 확인이 악성 파일 실행을 막는 첫 번째 방어선이 됩니다.

다만 확장자를 볼 수 있다고 해서 모든 파일의 안전성을 판단할 수 있는 것은 아닙니다. 정상적인 문서 확장자라도 출처가 불분명하면 위험할 수 있습니다. 확장자 확인은 출처 확인, 보낸 사람 확인, 파일을 요청한 맥락 확인과 함께 해야 의미가 있습니다.
