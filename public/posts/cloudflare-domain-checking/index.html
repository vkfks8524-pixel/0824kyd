Cloudflare에서 도메인이 확인 중으로 오래 남을 때 점검할 항목

작성일: 2026년 7월 12일 · 예상 읽기 시간: 8분

Cloudflare Pages에 도메인을 연결했는데 “확인 중” 상태가 오래 보이면 불안할 수 있습니다. 하지만 대부분은 네임서버 전파, DNS 레코드, SSL 인증서 확인 과정에서 생기는 정상 대기입니다.

1. 네임서버가 Cloudflare로 바뀌었는지 확인

도메인을 구매한 업체에서 Cloudflare가 안내한 네임서버 두 개를 정확히 입력했는지 확인합니다. 기존 네임서버가 함께 남아 있으면 확인이 늦어질 수 있습니다.

2. Pages 사용자 지정 도메인에서 추가했는지 확인

Cloudflare DNS 메뉴에서 레코드만 수동으로 만든 것과 Pages 프로젝트의 사용자 지정 도메인 메뉴에서 연결한 것은 다릅니다. Pages 프로젝트 안에서 도메인을 추가해야 인증이 정상적으로 진행됩니다.

3. 루트 도메인과 www를 구분하기

kyd.kr과 www.kyd.kr은 서로 다른 호스트입니다. 둘 다 사용할 계획이면 둘 다 사용자 지정 도메인에 추가하고, 대표 주소를 하나로 정해 리디렉션을 설정합니다.

4. SSL 인증서 발급 기다리기

DNS가 맞아도 HTTPS 인증서가 준비되기까지 시간이 걸릴 수 있습니다. 이때 도메인을 삭제하고 다시 추가하기보다 상태가 바뀌는지 기다리는 편이 낫습니다.

5. 24시간 후에도 그대로일 때

하루가 지나도 확인 중이라면 네임서버 오타, DNSSEC 설정, 잘못된 CNAME, 다른 서비스와의 충돌을 확인합니다. 특히 DNSSEC를 사용하던 도메인은 DS 레코드가 남아 있는지 볼 필요가 있습니다.

정리

확인 중 상태는 무조건 오류가 아닙니다. 네임서버, Pages 연결 위치, 루트와 www 구분, SSL 인증서, DNSSEC 순서로 차분히 확인하면 대부분 해결 방향을 찾을 수 있습니다.

확인 중 상태의 의미

Cloudflare에서 도메인이 확인 중으로 표시되는 것은 보통 DNS 전파, SSL 인증서 발급, Pages 연결 확인이 아직 끝나지 않았다는 뜻입니다. 설정이 잘못되었을 수도 있지만, 설정 직후라면 단순 대기 상태인 경우가 더 많습니다.

중요한 것은 기다리는 동안 같은 설정을 반복해서 지우고 다시 만드는 일을 피하는 것입니다. DNS는 전 세계 서버에 순차적으로 반영되기 때문에 중간에 계속 수정하면 오히려 확인 시간이 길어질 수 있습니다.

먼저 확인할 항목

도메인 구매처의 네임서버가 Cloudflare에서 제시한 두 개의 네임서버와 정확히 일치하는지 확인합니다. 철자가 하나라도 다르면 Cloudflare가 도메인을 관리할 수 없습니다. 예전 네임서버가 함께 남아 있는지도 확인합니다.

그 다음 Cloudflare Pages의 사용자 지정 도메인 화면에서 DNS 설정 완료 표시가 있는지 봅니다. DNS 설정은 완료되었는데 상태만 확인 중이라면 SSL 인증서나 전파 대기일 수 있습니다. 이 경우에는 몇 시간 기다리는 것이 정상적인 대응입니다.

하루가 지나도 그대로일 때

24시간 이상 확인 중이면 DNSSEC 또는 DS 레코드가 남아 있는지 확인합니다. 도메인 구매처에서 DNSSEC가 켜져 있으면 Cloudflare 네임서버로 바꿔도 검증이 실패할 수 있습니다.

또한 루트 도메인과 www 도메인을 동시에 연결했다면 각각의 상태를 따로 확인해야 합니다. 하나만 활성이고 다른 하나가 대기 중일 수 있습니다. 이때는 이미 활성화된 주소로 사이트가 열리는지 먼저 확인하고, 나머지 주소의 설정을 천천히 점검합니다.

기다리는 동안 확인할 수 있는 것

도메인 상태가 확인 중이어도 Pages의 기본 주소는 이미 열릴 수 있습니다. 예를 들어 pages.dev 주소에서 사이트가 정상적으로 보인다면 사이트 파일 자체는 배포된 것이고, 남은 문제는 사용자 지정 도메인 연결 쪽으로 좁혀볼 수 있습니다.

또한 DNS 레코드가 자동으로 만들어졌는지 확인할 수 있습니다. Pages 사용자 지정 도메인 화면에서 DNS 설정 완료가 보인다면 CNAME이나 라우팅 설정은 대체로 맞고, 네임서버 전파나 인증서 확인을 기다리는 단계일 가능성이 큽니다.

수정하면 안 되는 타이밍

설정 직후 10분이나 30분 정도 확인 중이라고 해서 도메인을 삭제하고 다시 추가하는 것은 좋지 않습니다. DNS 전파는 시간이 걸릴 수 있고, 중간에 설정을 바꾸면 확인 과정이 처음부터 다시 시작될 수 있습니다.

대신 변경한 내용을 기록하고 일정 시간 뒤 다시 확인합니다. 네임서버가 맞고 DNSSEC 문제가 없다면 대개 시간이 지나며 활성화됩니다. 하루가 지나도 그대로일 때만 구매처 설정과 Cloudflare 설정을 다시 차분히 비교하는 것이 좋습니다.
